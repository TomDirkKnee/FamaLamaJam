## Manual-Only Verification

The phase should close with a short Ableton export matrix covering:

- record toggle is blocked until a folder is chosen
- local interval stem starts on the next interval boundary and imports cleanly in a DAW
- local and remote voice-mode sources do not create or extend stem files
- switching a source back from voice mode to interval mode resumes stem capture only on the next interval boundary
- the explicit new-folder control creates a fresh recording-run folder without reconnecting and preserves the prior run's files
- disconnect/reconnect stays in the same session folder without corrupting earlier files
- turning recording off closes files on the next interval boundary

Planned artifact:
- `docs/validation/phase8.3.3-stem-capture-matrix.md`

## Decision Gate

Phase 08.3.3 is complete only if:
- focused automation is green on `build-vs-2026`
- the debug VST3 is rebuilt successfully
- manual Ableton validation confirms interval stems remain practically importable and aligned enough to mix without ad hoc nudging
- manual Ableton validation confirms voice chat is excluded from export entirely
- the explicit new-folder workflow creates a fresh session folder without relying on reconnect behavior

If the export path works technically but leaves unclear naming, alignment, or reconnect behavior, the closeout must say so explicitly rather than treating the phase as done.
