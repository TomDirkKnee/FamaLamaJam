# Phase 08.3.3 Context: Stem Capture & Export

## Goal

Add optional on-disk stem capture for local and remote recordings so users can import session stems into a DAW later without having to manually realign interval timing.

## Locked Decisions

### Stem scope

- Save both local and remote stems.

### Timing alignment

- Stem files must start on interval boundaries so later DAW import stays easy to line up.
- If recording is enabled mid-session, new files start on the next interval boundary.
- Do not pad leading silence before that boundary.
- If a remote user joins later, their file starts at their first actual audio with no leading silence before it.
- Local interval stems contain the actual transmitted interval contribution, not the raw live monitor input.
- Local voice stems also begin on the next boundary, then record continuous voice chunks from there.

### Format and signal

- Export WAV only in this phase.
- Export dry / pre-mix stems, not post-strip monitor output.
- Keep the real source channel format at export time rather than forcing stereo or mono.
- Remote interval and voice material are written as separate files.
- Append mode in the filename only when the file is voice mode.

### Folders and naming

- Create one new session folder per recording run.
- Name the session folder from BPM/BPI plus timestamp.
- Name each stem file with username plus channel name.
- Append mode only for voice files.
- Add a numeric suffix on filename collisions rather than overwriting.

### Record workflow

- Use one global `Record stems` control plus a folder picker.
- If the user arms recording before choosing a folder, block recording and show an inline error.
- If recording remains armed across disconnect/reconnect, keep the same session folder and restart any new files on the next interval boundary after reconnect.
- If the user turns recording off during a live session, stop and close files on the next interval boundary.

## Explicitly Deferred

- Multiple per-source recording toggles.
- Alternate export formats such as FLAC.
- Post-mix or dual dry/post exports.
- Timeline metadata or silence backfill for late joiners.

## Open Questions For Planning

- Where the `Record stems` toggle and folder picker should sit in the current UI without crowding the compact session flow.
- Whether folder selection should reuse JUCE-native file chooser UX directly or wrap it in additional validation state.
- How to cut voice-mode files cleanly at stop boundaries while preserving the interval-boundary start rule.
