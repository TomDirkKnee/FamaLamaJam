# Plan 08.3.3-05 Summary

## Outcome

Added an explicit in-session rollover control so the user can start a fresh recording run folder without reconnecting.

## What Changed

- processor now supports a boundary-queued `new stem folder` request
- the editor exposes a compact `New stem folder` control beside the existing stem-capture controls
- the UI shows pending rollover status instead of silently switching folders
- rollover keeps the chosen base export directory but starts a fresh session folder on the next interval boundary
- earlier files remain in the prior folder and reconnect continuity still reuses the current run when no rollover is requested
