# Phase 08.3.2 Plan 01: Mixed-Room Voice Transport Foundation Summary

**FLJ now has the first true mixed-room voice receive path: remote voice chunks are decoded and mixed immediately instead of waiting for interval promotion, while interval peers remain boundary-quantized.**

## Accomplishments

- Added mixed-room RED transport coverage and per-peer `channelFlags` support to the MiniNinjamServer harness.
- Added a focused `plugin_voice_mode_transport` integration suite that proves:
  - remote voice audio can be heard before the next interval boundary
  - interval peers in the same room still wait for their proper boundary
- Implemented a dedicated remote-voice playback queue in the processor so voice chunks bypass the interval scheduler entirely.
- Updated diagnostics and strip runtime state so voice peers stay visible and report as voice rather than looking like disappearing interval peers.
