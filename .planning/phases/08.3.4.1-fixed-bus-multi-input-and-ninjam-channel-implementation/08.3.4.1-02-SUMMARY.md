## Decisions Made

- Kept the local multi-input implementation on the existing upload transport path so the fixed-bus phase did not create a second sender architecture.
- Treated remote routing as a fixed-cap map keyed by stable remote source ID instead of extending the proof-only selected-source seam.
- Stored local visibility, local labels, and remote assignments by stable IDs so save/restore is not broken by user renames or temporarily absent strips.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Split input and output bus layout checks after expanding to three output buses**
- **Found during:** Task 1 (Implement fixed local slots and channel-indexed outbound uploads)
- **Issue:** The proof-era layout check assumed the same channel count logic for inputs and outputs, which broke second-local-input detection once the fixed extra output pairs were enabled.
- **Fix:** Added separate input and output layout validation helpers so the processor can accept the two-input fixed-bus host layout while still exposing three output buses.
- **Files modified:** `src/plugin/FamaLamaJamAudioProcessor.cpp`, `src/plugin/FamaLamaJamAudioProcessor.h`
- **Verification:** `build-vs-2026/tests/Debug/famalamajam_tests.exe "[plugin_host_multi_io_routing],[plugin_voice_mode_transport]"`
- **Committed in:** `eaf99a3`

**2. [Rule 1 - Bug] Persist remote output assignments outside transient strip snapshots**
- **Found during:** Task 3 (Wire the current editor and state model to the fixed routing runtime)
- **Issue:** Remote route choices could be lost on restore if a remote strip snapshot was absent when state was saved.
- **Fix:** Added a dedicated `remoteOutputAssignments` state tree keyed by source ID and restored it separately from strip snapshots.
- **Files modified:** `src/plugin/FamaLamaJamAudioProcessor.cpp`, `src/plugin/FamaLamaJamAudioProcessor.h`
- **Verification:** `build-vs-2026/tests/Debug/famalamajam_tests.exe "[plugin_mixer_ui],[plugin_state_roundtrip],[plugin_voice_mode_toggle]"`
- **Committed in:** `d505451`
