## Sampling Rate

- After every code-producing task commit, run the narrowest relevant `famalamajam_tests.exe` tag set for that task.
- After Wave 4 completes, run the Wave 4 shell gate before moving to strip ergonomics work.
- After Wave 5 completes, run the Wave 5 strip gate before preparing the final host rerun.
- Before handing off the `09.1-06` checkpoint, run the Wave 6 follow-up gate first, then run the Debug VST3 rebuild as a separate pre-checkpoint artifact step.
- After the human checkpoint response, run the Task `09.1-06-03` doc/state audit immediately.
- Before closing the phase after an approved rerun, run `ctest --test-dir build-vs-2026 --output-on-failure -C Debug`.

## Feedback Latency

- **Max Nyquist latency:** `~35s` because every automated task gate stays on test-only commands.
- **Checkpoint-prep latency:** the `09.1-06` Debug VST3 rebuild is tracked separately from Nyquist sampling because it is required for the host rerun artifact, not for fast test feedback.
- **Final sign-off latency:** `ctest` plus the blocking Ableton checkpoint. Human host time is intentionally excluded from Nyquist latency.
