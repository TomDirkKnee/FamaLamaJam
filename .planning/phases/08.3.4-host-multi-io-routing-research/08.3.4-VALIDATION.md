  - `P8.3.4-ROUTE-02` covers Ableton routed extra-output exposure

  - `P8.3.4-ROUTE-04` records the ergonomics verdict and fallback decision

## Fallback Decision Contract

- If Ableton does not expose the extra input in a normal sidechain or aux-input path, record the exact blocker and recommend `no follow-on implementation phase`.
- If routed output pairs are too hidden, too fragile, or not isolated enough for per-channel use, record the exact blocker and recommend `per-user` output routing as the fallback instead of expanding the fixed-bus scope.
- If the overall workflow is too awkward or too hidden for normal use, record the ergonomics blocker directly in the matrix and recommend either `per-user` output routing or `no follow-on implementation phase`.
- The manual result must never imply broader routing readiness beyond the fixed-bus proof.
- Approved outcome for this phase: Ableton exposed both proof buses in the normal host routing UI, so the research conclusion is to pursue a follow-on fixed-bus implementation phase and let Live own track-to-bus routing externally.
- The approved implementation direction is `Main` plus `Local Send 2` for local inputs and fixed extra output pairs for routed remote returns.
- Do not treat plugin-owned enumeration of Ableton tracks inside FLJ as the target architecture.
